# 数据结构

数据结构即'数据 + 结构'，描述了数据在物理结构和逻辑结构中的分布关系。

## 数据 Data

| 名称     | 含义                               |
| -------- | ---------------------------------- |
| 数据对象 | 性质相同的数据元素的集合           |
| 数据元素 | 有一定意义的数据基本单元           |
| 数据项   | 数据的逻辑存储的最小单位           |
| 数据     | 描述客观事物的符号，可被计算机操作 |

## 结构 Structure

- 物理结构: 数据的逻辑结构在计算机中的存储形式

  - 顺序存储结构（Sequence）：数据元素存放在连续的存储单元中。优点是可以实现随机访问，缺点是空间利用率不高，且插入和删除等逻辑操作消耗性能
  - 链式存储结构（linked）：数据元素存放在任意的存储单元中，每个元素包含数据和指向下一个元素的指针。优点是空间利用率高、插入和删除等逻辑操作效率高，但访问速度较慢。

- 逻辑结构：数据对象中数据元素之间的关系。
  - 集合结构（Set）：数据元素之间没有关系，每个元素都是独立的。
  - 线性结构（Linear）：数据元素之间是一对一的关系，如数组、链表、栈和队列。
  - 树形结构（Tree）：数据元素之间是一对多的关系，如二叉树、平衡树等。
  - 图形结构（Graph）：数据元素之间是多对多的关系，如无向图和有向图。

## ADT

使用一个数学模型以及在该模型之上的一组操作，来定义一个抽象数据类型(abstract data type,ADT)。

```ADT
Name 抽象数据类型的名字
Data
   数据元素之间逻辑关系的定义
Operation
   操作名
      初始条件
      操作结果描述
```

## QA

- 连续存储的数据，使用哪种数据类型最合适？

从性能上考虑，应该选择'对象数据类型'而不是'数组数据类型'，虽然后者已经实现接口了。分析：

- 添删操作：复杂度：对象 O(1)，数组在头部为 O(n) ，因为数组元素的整体向后移动会耗费性能
- 查询操作：当传入位置参数时，都是 O(1);

综上选择对象类型，算法复杂度低，内存的写操作更少。
